### 3. METHODOLOGY (500–700 words)

Organize into **4 subsections**.

#### 3.1 Problem Formulation

**One paragraph covering:**
- Binary classification task: bonafide (y=0) vs. spoof (y=1)
- Input: variable-length audio waveform
- Output: a detection score (lower = more likely bonafide)
- Decision threshold tuned to minimize EER
- Formally: f: x → s, where s is a real-valued score

#### 3.2 Feature Extraction

**What to cover:**
- Two streams: (a) raw waveform → sinc filters [4], (b) SSL features from wav2vec 2.0 [7]
- For SSL: which layer to extract from (cite [8] justification — intermediate layers best)
- Spectrogram as optional third input
- Preprocessing: 4-second fixed-length segments (pad/truncate)
- Sample rate: 16 kHz (ASVspoof 2019 standard)

**Include a simple pipeline diagram in text:**
```
Raw Waveform (16kHz)
       │
       ├──→ Sinc Filters → Residual Blocks → [End-to-End Stream]
       │
       └──→ wav2vec 2.0 (frozen/fine-tuned) → [SSL Stream]
                                                    │
                                      Graph Attention Network (AASIST)
                                                    │
                                          Detection Score
```

#### 3.3 Model Architecture

**Write a paragraph for each model you're implementing:**

**Baseline: RawNet2 [4]**
> "RawNet2 processes raw waveforms through a bank of sinc filters parameterized by learnable lower and upper cutoff frequencies, followed by six residual blocks equipped with Feature Map Scaling (FMS) attention. A GRU layer captures temporal dependencies before a fully-connected output layer produces a detection score. The architecture was originally designed for speaker verification and adapted for anti-spoofing by [cite RawNet2 paper]."

**Proposed: AASIST [5]**
> "AASIST constructs a heterogeneous graph where spectral and temporal artefacts form separate graph branches processed by stacked graph attention layers (HS-GALs). A heterogeneous attention mechanism and stack node integrate cross-domain artefacts, while a max graph operation introduces competitive selection among nodes. Even the lightweight AASIST-L variant (85K parameters) outperforms ensemble systems [5]."

#### 3.4 Data Augmentation

**Cover RawBoost [9] in one paragraph:**
> "To improve robustness to channel effects and real-world degradation, we apply RawBoost [9] during training. RawBoost operates directly on raw waveforms, applying three stochastic noise modes: (i) linear and non-linear convolutive noise modelling transmission channel effects, (ii) impulsive signal-dependent additive noise, and (iii) stationary signal-independent additive noise. Unlike SpecAugment, RawBoost requires no external noise recordings and is model-agnostic, making it directly compatible with both RawNet2 and AASIST. Experiments by [9] demonstrated a 27% relative improvement on ASVspoof 2021 LA when applied to a RawNet2 baseline."

**Loss function:**
> "Models are trained using weighted binary cross-entropy to handle the class imbalance in the training set, where spoofed utterances outnumber bonafide by approximately 9:1. The Adam optimizer is used with an initial learning rate of 1e-4, decayed by a factor of 0.1 after plateau detection. Mini-batches of 32 utterances are used, with each waveform randomly cropped to 64,000 samples (4 seconds at 16kHz)."

## Markdown Formatting Rules for Clean PDF Rendering

```markdown
# Level 1: Report Title only
## Level 2: Major sections (Abstract, 1. Introduction, etc.)
### Level 3: Subsections (2.1, 2.2, etc.)
#### Level 4: Sub-subsections only if necessary

**Bold** for key terms on first use
*Italic* for paper names or datasets
`Code font` for variable names, metrics, code
> Blockquote for direct quotations
```

**Tables:** Always include header row with `|---|` separator.

**Equations:** Use `$...$` for inline math, `$$...$$` for display math:
```
EER is defined at the threshold $\theta^*$ where:
$$\text{FAR}(\theta^*) = \text{FRR}(\theta^*)$$
```

**Figures:** Use ASCII diagrams for architecture; describe any actual figures with `[Figure X: caption]` as placeholder.
